  • Why are there diminishing returns at higher concurrency levels?

We can observe from -c 32 and above the average request stopped doubling. I think the reason is that although mutiple requests are sent at a time, the github page has a limited processing time for each request. Hence the maximum processing time is about 180~ [#/sec] regardless the number of request recieved.
